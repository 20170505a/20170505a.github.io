什么是SRA

SRA的全称是Short Read Archive，它是由NCBI提供的一个数据储存服务。从这个名称上来看，貌似是储存测序的原始数据的一个数据库。SRA中的数据组织形式是按照层级结构组织的，不用的数据前面有不同的前缀，如下所示：

  1. NCBI的生物项目(BioProject)的命名为PRJN****，例如PRJNA257197，这个数据是对一个单独研究机制的整体描述，一个生物项目含有很多样本与数据集；
  2. NCBI的生物样本(BioSample)，它的命名为SAM****，或者是SRS****，例如SAMN03254300，这个数据指的是生物样本的来源，每个生物学样本都有一个单独的BioSample编号，以及一组独特的特征。
  3. SRA实验，命名方式为SRX****，这是针对某个特定样本的一个测序文库。
  4. SRA run，命名方式为SRR****或ERR****，例如SRR1553610，它对应的是某个测序文库的数据文件。

SRA中的spot

当我们在SRA中检索时，会发现这样的信息，如下所示

mark

SRA中呈现的结果是Spots，而非reads，其实这两个术语描述的是一种东西。但是如果真要区分这两个东西，可以从Illumina测序的原理上来理解，在测试时，最终的序列是靠相机来读取的，一个spot上有一串碱基，，spot就是这串碱基（就是桥式PCR一端的结合位点）所在的位置，因此spot中包括了这些信息：接头，测序引物，目标序列片段，index，这是从技术层面来讲的，如果从生物学层面来讲，我们所说的reads其实就是指的目标序列，是去掉了接头，测序引物，index后的序列信息。但在很多情况下，spots跟reads所表达的意思是一样的。

xargs

xargs命令的功能是：将上游输入的数据转化为特定的命令，以上面的命令为例解释一下cat ids.txt | xargs -n 1 echo This is number $1，这个命令是将ids.txt中的SRR号作为echo This is number $1的输入，由于idx。txt这个文件中一共有10行，那么这个-n 1此时就表示，把每一行的信息作为参数传递给THis is number $1，如果我们把-n 1改为-n 2，那么就是下面的情况：

mark

可以发现，一次传递了2个参数的内容。

xargs后面还有东西，也就是echo This is number $1，这个echo好理解，把它当成输出某些信息的函数即可，这个$1表示脚本运行中的第1个参数，具体的内容参考以前的笔记《Shell学习笔记（4）——处理用户输入》。
